## 登录流程（浏览器）

XHS 不再支持单个二维码扫描。始终使用浏览器登录二进制文件：

### 当 `check_login_status` 返回 ❌ 未登录:

**Step 1: 运行浏览器登录二进制文件**
```bash
cd ~/xiaohongshu-yuzhou && ./xiaohongshu-login-darwin-arm64 -bin "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
```
通过 Bash 工具运行。它会打开一个 Chrome 浏览器窗口。

**Step 2: 告诉用户登录**
告诉用户："浏览器已打开，请在 Chrome 窗口中完成小红书登录（可能需要扫码两次），完成后告诉我。"

**Step 3: 等待用户确认**
在用户确认成功登录之前不要继续。

**Step 4: 重启 MCP server**
```bash
cd ~/xiaohongshu-yuzhou && bash start_mcp.sh
```

**Step 5: 验证登录**
再次调用 `check_login_status` 确认 ✅ 已登录。

### ❌ 切勿使用 `get_login_qrcode`
二维码登录工具已失效 — XHS 现在需要扫码两次，该工具无法处理。

## 常见错误

### ❌ 跳过用户审核
**不好：** 生成内容 → 立即发布
**正确：** 生成内容 → 呈现审核 → 等待批准 → 发布

### ❌ 风格不匹配
**不好：** 命令式、教科书式语言
**正确：** 温柔、日记式、第一人称、鼓励语气

### ❌ 标题长度不对
**不好：** 50+ 字的长标题
**正确：** ≤20 字，温暖、有吸引力

### ❌ 正文过长
**不好：** 800+ 字的冗长文章
**正确：** 300-450 字，精准、节奏清晰

### ❌ 没有福气钩子
**不好：** 纯粹分享知识
**正确：** 每篇文章都有互动吸引（"刷到这篇的人…"、"留言的人…"）

### ❌ 图片拥挤
**不好：** 大量文字叠加在图片上
**正确：** 纯净底图，温暖配色，简洁留白

### ❌ 违反宇宙能量风格
**不好：** 冷调、复杂、密集的视觉设计
**正确：** 温暖、梦幻、柔和、舒适感
